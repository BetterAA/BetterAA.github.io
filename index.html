JAVA hashMap and treeMap

这里简单介绍下这两个map

也是一些我从网上收集的内容，有些测过有些没测，可能有不对的地方 ：D

关于hashMap和 treeMap

HashMap通过hashcode对其内容进行快速查找，而 TreeMap中所有的元素都保持着某种固定的顺序（内部实现应该是红黑树），如果你需要得到一个有序的结果你就应该使用TreeMap（HashMap中元素的排列顺序是不固定的）。

性能对比：

两种常规Map性能
HashMap：适用于在Map中插入、删除和定位元素。
Treemap：适用于按自然顺序或自定义顺序遍历键(key)。

JAVA异常：Comparison method violates its general contract

异常描述

java.lang.IllegalArgumentException: Comparison method violates its general contract!

异常举例

Collections.sort(list, new Comparator<Integer>() {  
@Override  
    public int compare(Integer o1, Integer o2) {  
        return o1 > o2 ? 1 : -1;// 错误的方式  
    }  
});

异常原因

JDK7的sort函数实现变了，与旧版本的不同，存在兼容问题，可以查看下自己的compare函数实现有木有问题

Hadoop hive 分区partition

Hive是啥子？

  1. Hive是基于Hadoop的一个数据仓库工具，可以将结构化的数据文件映射为一张数据库表，并提供类SQL查询功能。（将输入的hql语句解析成待提交的job）
  2. 本质是将SQL转换为MapReduce程序

Hive 特点

  1. 可扩展： 可自由扩展集群规模，一般无需重启服务
  2. 延展性： 支持自定义函数
  3. 容错性： 良好的容错性，节点出现问题SQL仍可完成
