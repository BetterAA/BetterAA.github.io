docker 使用小记

  • docker

最近看师兄笔试安全工程师的题有很多考docker的，这两天来安装docker顺便来体验下。

依旧是简介：docker是什么

  1. 开源引擎
  2. 可以轻松为任何为任何应用创建一个轻量级、可移植、自给自足的容器
  3. 容器可以批量在生产环境中部署，包括 虚拟机、OpenStack集群

上手体验过程

1. 安装下载

建议从官网下载安装包，这样能保证不会出现奇怪的问题

mac版地址

windows版地址
我的百度云盘上存了备份

linux版的地址

安装完毕之后 docker version 查看安装情况

2. 搜索可用的镜像

docker search xxx
这里以tutorial镜像为例

会找到不同用户空间下的名为tutorial的镜像，在docker的镜像索引网站上，镜像都是按照 用户名/镜像名 的方式来存储的。除了部分 像ubuntu这类基础镜像，经过官方验证，可信，可以直接用镜像名来检索到
（这里的用户指的是hub.docker.io官方的注册用户，建议同时也在docker官网上注册下，否则docker login时候会出问题）

3. 下载镜像

根据镜像的存储方式，用docker pull 即可。(和git类似)

4. 在docker容器中运行hello world

docker run命令可以启动某个镜像，并运行一个命令
注： 【docker容器】可以理解为在沙盒中运行的【进程】，但是沙盒默认不会运行任何程序，你需要在沙盒中运行一个进程来启动某个容器。 这个进程是该容器的【唯一进程】

5. 在容器中安装新的程序

例子中要给之前pull的learn/tutorial镜像安装一个ping程序，因为tutorial是基于ubuntu的，因此可以通过apt-get来按照

6. 保存对容器的修改

当对一个容器修改之后（例如上文docker run运行一个命令），可以把修改保存下来，下次从新版本上继续

从此就有了一个新的镜像，名为betteraa/tutorial ，可以和上面一个docker run在里面运行新的命令

7. 查看运行中的镜像

8. 发布修改后的镜像

步骤： 查看已有的镜像信息-》登录docker账号-》发布指定镜像

上文是个简单的过程介绍，是先用一个已有的镜像，然后修改commit成一个新的镜像后发布，其实也可以自己制作镜像然后发布。

下面介绍个简单的通过Dockerfile制作镜像的过程

当你在使用 Docker构建镜像的时候，每一个命令都会在前一个命令的基础上形成一个新层

通过Dockerfile自动构建容器的过程 其实就相当于上文把每个docker run的命令打包一起写到 一个Dockerfile文件中， 不过Dockerfile也有自己的一套简单的语法

Docker语法包含两部分，注释# + 命令
这里先列举几个常用的，然后附上一个Dockerfile文件作为参考。

FROM ubuntu:14.10 使用哪个镜像作为基础镜像启动构建流程
MAINTAINER xxx 定义作者
RUN   用于【创建镜像】的命令，在之前commit层之上形成新的层
CMD   和run不同的是，这些命令不是在镜像构建过程中执行的，而是在镜像构建容器之后被调用
ENV  SERVER_WORK 4 设置环境变量，以key value形式定义
ADD命令有两个参数，源和目标。它的基本作用是从源系统的文件系统上复制文件到目标容器的文件系统。如果源是一个URL，那该URL的内容将被下载并复制到容器中。

示例： 创建一个Nginx的镜像

vim Dockerfile
2. 编辑Dockerfile文件

设置命令与参数下载Nginx

安装nginx之后，需要配置Nginx并且替换掉其默认的配置文件

保存Dockerfile文件
这里ADD命令用到了一个nginx配置文件 ngnix.conf

vim ngnix.conf

具体配置如下：

worker_processes 1;
events { worker_connections 1024; }
http {
     sendfile on;
         server {
         listen 80;
         location / {
              proxy_pass http://httpstat.us/;
              proxy_set_header X-Real-IP $remote_addr;
         }
     }
}

保存ngnix.conf

关于docker镜像的清理与删除

关于docker仓库的镜像的删除
其实感觉每个镜像都挺大的，因此清理下冗余的镜像也是有必要的

1. 停止所有的container，这样才能删除其中的images
2. 删除容器与删除镜像

删除容器 docker rm +id
删除全部容器 docker rm $(docker ps -a -q)
删除镜像 docker rmi （image id) #通过docker images获得镜像id

删除全部镜像 docker rmi $(docker images -q)
（【docker容器】可以理解为在沙盒中运行的【进程】）

docker的本地私有仓库搭建

和Maven一样， docker不仅提供了一个中央仓库（hub.docekr.io），同时也允许registry搭建本地私有仓库（就像本地git仓库）
好处：

  1. 节省带宽， 比中央仓库快很多。
  2. 针对公司内部使用的镜像，可以供公司内部使用
