chrome中 页面 自动点击功能实现（console版，插件版）

  • chrome插件

背景： 比较好奇这个功能，今天特意来写写，真的是好久不写js忘的一干二净 ；P

测试场景：

自己写了一个测试界面，点击这里
需要点击三次之后才能调到后台处理,模拟需要多次点击场景

方法一： 直接在chrome的console运行代码

F12打开检查元素页面，点击source标签，可以直接在下面的console中写js代码在当前页面执行，例如

//循环alert 5次
for(var i=0; i<5;i++) alert(i);

不过这样刷新页面后就没有了，也可以在source标签下面的snippets标签中new一个临时脚本，把上面代码放进去，以后仍然可以继续访问，右键选择evaluate in console同样在console中运行

方法二：chrome插件版本

chrome运行加载自己开发的插件，不过需要按照一定的格式，其中包括配置文件manifest.json, 插件图标,以及自己的脚本文件。

先新建一个文件夹，用来放置需要的文件，在文件夹中：

1. 新建一个文本文件 manifest.json，内容如下：

{
      "name": "第一个Chrome插件",
    "manifest_version": 2,
    "version": "0.0.1",
    "description": "第一个Chrome插件,自动点击",
    "browser_action": {
    "default_icon": "1.png"
    },
    "content_scripts": [
    {
        "matches": ["http://123.206.23.200:5000/index"],
        "js": ["myscript.js","jquery.js"]
    }]        
}
}

解释一下最后content_scripts，matches代表在只匹配到的页面中加载这个插件，这里设置为我们的测试页面，可以用正则改为所有页面中都使用 [“http://*/*","https://*/*“]
js代表我们需要加载的自己用的脚本文件，这里myscript是自己的脚本，顺便把jquery也加进去了
(对了，记得放一张1.png文件作为图标)

2. myscript.js 内容如下：

setInterval(function () {document.forms[0].submitBtn.click(); },1800);

意思是每隔1.8s 触发一次提交动作。这里是在检查了测试页面的提交方法之后写的

3. 测试

在chrome中，打开chrome://extensions/， 点击 加载已解压的扩展程序 ，选择刚刚新建的文件夹，确定之后就加载成功了，工具栏中可以看到新的图标。 打开测试页面进行测试吧 ：）

每次修改扩展程序之后可以点击重新加载，最后测试完成之后可以 打包成crx文件。
其实crx文件是个压缩文件，可以改下后缀解压。 复杂点的功能可以下一个现成的crx解压打开看下学习
